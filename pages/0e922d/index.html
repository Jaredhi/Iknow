docker实战

# 前言

# 关于容器挂载路径

我一般存放到 /home/username/docker/ 下面，username 是你服务器上的用户名称。

对应挂载的文件目录应该提前创建好。例如：

mkdir -p /home/jared/nginx/{conf,conf.d,html,logs}
1

# 关于容器内部时间

docker容器内时间与服务器时间差距8小时，因为原生docker是0时区，但我们服务器采用亚洲上海东八区。

# 未启动容器

  • 将本地的 /etc/localtime 挂载到容器中 -v /etc/localtime:/etc/localtime

    -v /etc/localtime:/etc/localtime
    1

  • （不一定需要）timezone

    -v /etc/timezone:/etc/timezone
====================
# 以下是文件内容
Asia/Shanghai
    1
    2
    3
    4

# 已经启动容器

  1. 首先进入容器 docker exec -it /bin/sh 查询时间 date -R

  2. 创建文件夹mkdir -p /usr/share/zoneinfo/Asia

  3. 回到宿主机，复制宿主机里的文件到容器中

    docker cp /usr/share/zoneinfo/Asia/Shanghai 容器ID或容器名:/usr/share/zoneinfo/Asia
    1

  4. 进入容器

    docker exec -it 容器Id或容器名
cp -i /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
    1
    2

    此时可以会报错cp: not writing through dangling symlink '/etc/localtime'

    解决方法：

    删除容器内的/etc/localtime,重新复制

    rm -rf /etc/localtime
cp -i /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
    1
    2

  5. date 查看时间

    在创建的时候最好填把 localtime 和 timezone 两个文件映射到容器内 /etc/localtime 和 /etc/timezone

# Nginx

# 挂载目录

mkdir -p /yourdir/nginx/{conf,conf.d,html,logs}

说明 本机 容器
配置文件 /opt/nginx/conf/nginx.conf /etc/nginx/nginx.conf
日志输出 /opt/nginx/logs /var/log/nginx
静态资源 /opt/nginx/html /usr/share/nginx/html

# 创建容器

注意：先将配置好的 nginx.conf 放到外部挂载的conf目录下

sudo docker run --name nginx -d -p 80:80  -v /opt/nginx/conf/nginx.conf:/etc/nginx/nginx.conf  -v /opt/nginx/logs:/var/log/nginx -v /opt/nginx/html:/usr/share/nginx/html -d nginx:1.18
1

# Jenkins

# 挂载目录

说明 本机 容器
工作目录 /home/yourname/docker/jenkins_home /var/jenkins_home
时区 /home/yourname/docker/timezone /etc/timezone
时间 /home/yourname/docker/localtime /etc/localtime

# 创建容器

sudo docker run --name jenkins_wiki  -p 10240:8080 -p 10241:50000 -v /home/yourname/docker/jenkins_home:/var/jenkins_home -v /home/yourname/docker/timezone:/etc/timezone -v /home/yourname/docker/localtime:/etc/localtime  -d jenkins/jenkins:lts
1

# 注意事项

报错 : 权限问题

Can not write to /var/jenkins_home/copy_reference_file.log. Wrong volume permissions?
touch: cannot touch ‘/var/jenkins_home/copy_reference_file.log’: Permission denied
1
2

原因是Jenkins镜像内部使用的用户是jenkons，但是我们启动容器时的账号是root，导致没有权限操作内部目录，我们可以稍微改一下上面的命令：

sudo docker run --name jenkins_wiki  -p 10240:8080 -p 10241:50000 -v /home/yourname/docker/jenkins_home:/var/jenkins_home -v /home/yourname/docker/timezone:/etc/timezone -v /home/yourname/docker/localtime:/etc/localtime -u 1001 -d jenkins/jenkins:lts
1

这里 -u 1001 的意思是覆盖容器内部的账号，引用 id为1001的账号

docker实战

# 前言

# 关于容器挂载路径

我一般存放到 /opt/docker/ 下面。（有 root 用户情况下）

对应挂载的文件目录应该提前创建好。例如：

mkdir -p /opt/nginx/{conf,conf.d,html,logs}
1

# 关于容器内部时间

docker容器内时间与服务器时间差距8小时，因为原生docker是0时区，但我们服务器采用亚洲上海东八区。

# 未启动容器

  • 将本地的 localtime和timezone ,挂载到容器中。

  • -v /etc/localtime:/etc/localtime

  • -v /etc/timezone:/etc/timezone （不一定需要）

    # 以下是 timezone 文件内容
Asia/Shanghai
    1
    2

# 已经启动容器

  1. 首先进入容器 docker exec -it /bin/sh 查询时间 date -R

  2. 创建文件夹mkdir -p /usr/share/zoneinfo/Asia

  3. 回到宿主机，复制宿主机里的文件到容器中

    docker cp /usr/share/zoneinfo/Asia/Shanghai 容器ID或容器名:/usr/share/zoneinfo/Asia
    1

  4. 进入容器

    docker exec -it 容器Id或容器名
cp -i /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
    1
    2

    此时可以会报错cp: not writing through dangling symlink '/etc/localtime'

    解决方法：

    删除容器内的/etc/localtime,重新复制

    rm -rf /etc/localtime
cp -i /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
    1
    2

  5. date 查看时间

    在创建的时候最好填把 localtime 和 timezone 两个文件映射到容器内 /etc/localtime 和 /etc/timezone

# Nginx

# 挂载目录

mkdir -p /opt/nginx/{conf,conf.d,html,logs}

说明 本机 容器
默认配置文件 /opt/nginx/conf/nginx.conf /etc/nginx/nginx.conf
其他配置 /opt/nginx/conf.d/ /etc/nginx/conf.d
日志输出 /opt/nginx/logs /var/log/nginx
静态资源 /opt/nginx/html /usr/share/nginx/html

# 创建容器

注意：先将配置好的 nginx.conf 放到外部挂载的conf目录下。🍌nginx 配置详解 (opens new window)

默认 nginx.conf 

user  nginx;
worker_processes  1;

error_log  /var/log/nginx/error.log warn;
pid        /var/run/nginx.pid;


events {
    worker_connections  1024;
}


http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    access_log  /var/log/nginx/access.log  main;

    sendfile        on;
    #tcp_nopush     on;

    keepalive_timeout  65;

    #gzip  on;

    include /etc/nginx/conf.d/*.conf;
}

# Jenkins

# 挂载目录

说明 本机 容器
工作目录 /opt/jenkins_home /var/jenkins_home
时区 /etc/timezone /etc/timezone
时间 /etc/localtime /etc/localtime

# 创建容器

sudo docker run --name jenkins_wiki \
-p 10240:8080 -p 10241:50000 \
-v /opt/jenkins_home:/var/jenkins_home \
-v /opt/timezone:/etc/timezone \
-v /opt/localtime:/etc/localtime  \
-d jenkins/jenkins:lts

# 注意事项

报错 : 权限问题

Can not write to /var/jenkins_home/copy_reference_file.log. Wrong volume permissions?
touch: cannot touch ‘/var/jenkins_home/copy_reference_file.log’: Permission denied
1
2

原因是Jenkins镜像内部使用的用户是jenkons，但是我们启动容器时的账号是root，导致没有权限操作内部目录，我们可以稍微改一下上面的命令：

sudo docker run --name jenkins_wiki  -p 10240:8080 \
-p 10241:50000 \
-v /opt/jenkins_home:/var/jenkins_home \
-v /etc/timezone:/etc/timezone \
-v /etc/localtime:/etc/localtime \
-u 1001 -d jenkins/jenkins:lts

# Redis

# 挂载目录

说明 本机 容器
配置文件目录 /opt/redis/conf /var/jenkins_home
数据目录 /opt/redis/data /etc/timezone

# 创建容器

sudo docker run --name myredis \
-p 6379:6379 --restart always -d --privileged=true \
-v /opt/redis/conf/redis.conf:/etc/redis/redis.conf \
-v /opt/redis/data:/data  \
redis redis-server /etc/redis/redis.conf \
--appendonly yes

# 说明

-d                                                  -> 以守护进程的方式启动容器
-p 6379:6379                                        -> 绑定宿主机端口
--name myredis                                      -> 指定容器名称
--restart always                                    -> 开机启动
--privileged=true                                   -> 提升容器内权限
-v /opt/redis/conf:/etc/redis/redis.conf    				-> 映射配置文件
-v /opt/redis/data:/data                    				-> 映射数据目录
--appendonly yes                                    -> 开启数据持久化
